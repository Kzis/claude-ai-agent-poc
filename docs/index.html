Architecture

Three Agents. One Pipeline.

Each agent has a distinct role and communicates through Notion as shared state, mirroring how real engineering teams work.

PM / PO Agent

Plan & Prioritize

Creates and prioritizes the product backlog, defines releases, and generates sprint reports — just like a real product owner.

notion.createTask notion.createRelease generateReport
SE Agent

Build & Ship

Picks the highest-priority task, creates a GitHub branch, writes production-quality code, opens a draft PR, and logs work to Notion.

github.createBranch github.openPR notion.logWork
QA Agent

Test & Verify

Reviews tasks in "In Review", runs test cases, files bug reports to Notion when issues are found, and generates a go/no-go release report.

notion.fileBug notion.approveTask generateQAReport
